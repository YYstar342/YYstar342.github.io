Markdown部署Hexo方法

1. 确保 Markdown 文件保存在正确的位置

Hexo 要求所有的文章 Markdown 文件必须保存在 source/_posts 目录中。例如：

  • 文件名：建议使用英文和短横线（-）分隔，避免使用空格或特殊字符。
  • 文件路径：确保文件路径正确，不要将文件保存到其他目录。

2. 使用 Typora 编写 Markdown 文件

  1. 打开 Typora。

  2. 编写你的文章内容。

  3. 在文件开头添加 Front Matter，用于定义文章的元数据（如标题、日期、分类等）。例如：

    markdown

    复制

    • **title**：文章标题。
    • **date**：文章发布日期（格式为 YYYY-MM-DD）。
    • **categories**：文章分类（可以是一个或多个）。
    • **tags**：文章标签（可以是一个或多个）。

5. 访问 GitHub Pages

  1. 打开浏览器，访问你的 GitHub Pages 地址（例如 https://YYstar342.github.io）。
  2. 如果文章显示正常，表示操作成功。

6. 检查生成的文件

如果文章仍然看不到，可以检查以下内容：

  1. 打开 Hexo 项目的 public 目录，检查是否有对应的 HTML 文件（例如 public/2023/10/01/my-first-post/index.html）。
  2. 如果没有生成对应的 HTML 文件，可能是以下原因：
    • Markdown 文件未正确保存到 source/_posts 目录。
    • Front Matter 格式错误。
    • 文章发布时间是未来时间（Hexo 默认不会生成未来文章）。

7. 启用未来文章生成（可选）

如果文章的发布时间是未来时间，可以在 _config.yml 中启用未来文章生成：

yaml

复制

8. 检查 GitHub Pages 设置

  1. 打开 GitHub 仓库页面（例如 https://github.com/YYstar342/YYstar342.github.io）。
  2. 点击 Settings 选项卡。
  3. 在左侧菜单中选择 Pages。
  4. 确保 Source 部分选择了正确的分支（通常是 main 或 master）。

总结

  1. 使用 Typora 编写 Markdown 文件，并保存到 source/_posts 目录。
  2. 确保文件包含正确的 Front Matter。
  3. 清除缓存并重新生成静态文件。
  4. 部署到 GitHub Pages。
  5. 访问 https://YYstar342.github.io 查看效果。
